Release Notes for Sencha Touch 2.0.pr2

Release Notes: Release Notes: November 10, 2011
Version Number: 2.0.0.pr2

New In This Release

  • Enhancements
    • Forms have been refactored to be less component heavy for better performance
    • Added Android native packaging guide
    • contentEl has been re-implemented on Components
    • Updated documentation for autoCreateViewport to clarify how it works
    • listeners and bubbleEvents config options and now guarenteed run prior to other configurations
    • [TOUCH-930] DataView and List now use proper getters when retrieving data from their Store
    • [TOUCH-775] Developer calls to Store.loadRecords will now fire 'clear' and 'add' events when appropriate
    • [TOUCH-831] Added a 'sort' event to Store
    • [TOUCH-842] Added a 'filter' event to Store
    • [TOUCH-908] Added emptyText support to DataView/List
    • [TOUCH-792] Completed Map class refactor
    • [TOUCH-793] Completed Geolocation class refactor
    • [TOUCH-928] Include jsb3 file in the distribution
    • [TOUCH-22] If a component's 'html' property is set, it will default 'styleHtmlContent' to true
    • [TOUCH-802] TextArea icon is now top aligned
    • [TOUCH-859] readOnly attribute added to fields
    • [TOUCH-927] Deprecate html in Button in favor of badgeText
  • Bugs Fixes
    • [TOUCH-763] Fix issue where SegmentedButton was not firing painted events
    • [TOUCH-763] Fix issue where TabBar was not firing painted events
    • [TOUCH-845] Fix persistent scrollbar issue in Safari 5.x desktop
    • [TOUCH-849] Fix issue with activeItem configuration not being respected

    • [TOUCH-841] Fix issue when inserting data into a store where List would throw an error
    • [TOUCH-931] Fix issues with multiselect and deselect within Selectable

    • [TOUCH-842] Fix issue when filtering a DataView/List by using the new 'filter' event on Store
    • [TOUCH-543] Fix issue where beforeselect event was not being fired on Selectable
    • [TOUCH-961] Fix issue with group headers not updating properly when the store changes
    • [TOUCH-893] Fix issue with multiple fields with the name inside a formpanel
    • [TOUCH-992] Fix setValue issue with numberfield and selectfield
    • [TOUCH-932] Fix issues with Ext.select
    • [TOUCH-836] Fix issue with Picker overflow which was causing alignment problems
    • [TOUCH-920] Aliased Ext.log to Ext.Logger.log to fix issues with shared platform code
    • [TOUCH-977] Fix segmented button styling issues

    • [TOUCH-832] Allow Carousel to be destroyed without throwing errors

    • [TOUCH-987] Fix issues related to List headers when sorting or otherwise updating data
    • [TOUCH-567] Fix form layout issue
    • [TOUCH-592] Fix issue with scrubber on Android 3
    • [TOUCH-654] Fix not being able to tap on a dropdown icon/arrow
    • [TOUCH-768] Fix issue when entering non-numeric values in numberfield
    • [TOUCH-800] Fix issue where the clearIcon was not properly centered on an iPad
    • [TOUCH-803] Fix issues with numberfield.reset()

    • Add missing constructor to example in class system guide
    • Fix Ext.Msg.alert call in class system guide
    • Fix error when destroying SizeMonitor detectors if the monitored element had already been removed
  • Known Issues
    • Updated MVC with Routing/History/Deep Linking/Profiles will be in a forthcoming release
    • The Data package currently cannot yet be configured by passing 'config' parameters. This will be fixed in an forthcoming release.
    • Draggable is not yet implemented yet, this affects carousel scrolling and animations
    • Some animations are still not in place and will be added in a forthcoming release
    • Android 3 and Blackberry optimizations are not yet in place
